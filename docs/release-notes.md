# Release Notes

Use this structure for each release.

## Version

- version number
- release date
- scope statement

## Doctrine

- changes that affect the constitution,
- naming or anti-pattern changes,
- recipe or contract changes that alter the grammar.

## Runtime

- foundation changes,
- primitive changes,
- component changes,
- recipe changes.

## Tooling and contract

- CLI changes,
- schema or validation changes,
- prompt or export changes.

## Docs and examples

- docs pages added or revised,
- new example coverage,
- demo or release-material changes.

## Risks and follow-up

- remaining sharp edges,
- intentionally deferred work,
- migration or compatibility notes.
