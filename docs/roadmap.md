# Roadmap

## V0

Theme: prove the doctrine through runtime, demos, CLI, and contract export.

Includes:

- constitution freeze,
- foundation runtime,
- core component set,
- editorial and SaaS demonstrators,
- CLI export and validation baseline.

## V0.5

Theme: tighten validation and deepen recipes without broadening the system excessively.

Includes:

- plan command maturity,
- anti-pattern validator expansion,
- more reference examples,
- sharper recipe adaptation rules.

## V1

Theme: broaden only after the doctrine is stable and legible.

Includes:

- additional workflow recipes,
- richer recipe browser or explorer,
- more formal agent tooling,
- controlled extension points.

## V2

Theme: deeper agent planning and carefully bounded extension mechanisms.

Includes:

- protocol-native adapters,
- recipe synthesis from constrained intent,
- conformance repair workflows,
- limited theme families that preserve the constitution.
